# MVP1-FEAT-001 ログイン機能

## 参照資料

* `login-flow.png` を画面デザインおよび状態遷移の参考資料とする

---

## 機能概要

Linknest 一般ユーザー向けUIにおけるログイン画面を実装する。

対象画面は以下とする。

* `/login`

---

## 実装対象

### 工程1: 初期表示の静的UI

WF内の「1. ログイン画面」を対象とし、以下の要素を表示する。

* シンボルロゴ
* タイトルロゴ
* キャッチコピー「つながりがチームを強くする」
* メールアドレス入力欄
* パスワード入力欄
* パスワード表示アイコン
* ログインボタン
* 「パスワードをお忘れですか？」リンク
* 区切り線と「または」
* 「アカウントをお持ちでない方は 新規登録」
* フッターリンク

  * 利用規約
  * プライバシーポリシー
  * サポート

### 工程2: 入力状態管理・パスワード表示切替・フォーカス表示

以下を実装する。

* メールアドレス入力値の状態管理
* パスワード入力値の状態管理
* パスワード表示／非表示切替
* 表示状態に応じた目アイコン切替
* 表示状態に応じた `aria-label` 切替
* 入力欄・ボタン・リンクのフォーカス表示対応

### 工程3: ログインボタン活性・非活性制御

以下を実装する。

* メールアドレスとパスワードの入力状態に応じたログインボタンの活性・非活性制御
* メールアドレスが1文字以上、かつパスワードが1文字以上入力されている場合のみログインボタンを活性化する
* 空白のみの入力では活性化しないよう、判定には `trim()` を使用する
* 非活性状態では `disabled` 属性を付与する
* 非活性状態のスタイルを追加する

## 実装方針

* Next.js App Router / TypeScript / CSS Modules を使用する
* CSSリセットには `destyle.css` を使用する
* 外部UIコンポーネントライブラリは追加しない
* 新しいnpmパッケージは追加しない
* CSS Modules のクラス名はBEM形式・ケバブケースで記述する
* レイアウトには原則 `display: grid` を使用する
* `display: flex` は使用しない
* 現段階では過剰なコンポーネント分割を行わない
* 実装前に、変更予定ファイルと実装方針を提示する

## 開発環境メモ

iPhone実機からLAN経由でNext.js開発サーバへアクセスする場合、`next.config.ts` の `allowedDevOrigins` に、iPhoneからアクセスするPCのLAN内IPを追加する。

例:

```ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  allowedDevOrigins: ["192.168.0.xxx"],
};

export default nextConfig;
```

`allowedDevOrigins` に指定するのは、iPhone自身のIPではなく、iPhoneのSafariで開くURLのホスト部分である。
